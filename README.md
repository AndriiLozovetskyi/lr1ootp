Розробити програмне забезпечення для відправлення повідомлень різними способами:

Email
SMS
Push-повідомлення

Створення об'єктів повідомлень повинно здійснюватися за допомогою патерна Factory Method.
На першому етапі було визначено задачу розробки системи відправлення повідомлень різними каналами зв'язку. Було встановлено, що система повинна підтримувати декілька типів повідомлень: Email, SMS та Push-повідомлення.

Оскільки всі типи повідомлень мають спільну функцію відправлення, але відрізняються реалізацією, було прийнято рішення використати породжувальний патерн проєктування Factory Method.

2. Створення базового класу Message

Для забезпечення єдиного інтерфейсу взаємодії було створено абстрактний клас Message, який містить віртуальний метод send(). Даний клас є базовим для всіх типів повідомлень.

class Message {
public:
    virtual void send(const string& text) = 0;
    virtual ~Message() {}
};
3. Реалізація конкретних типів повідомлень

На основі класу Message були створені класи:

EmailMessage;
SmsMessage;
PushMessage.

Кожен клас перевизначає метод send() відповідно до свого типу повідомлення.

Наприклад, клас EmailMessage виводить повідомлення про відправлення електронного листа, а SmsMessage — про відправлення SMS.

4. Створення абстрактної фабрики

Для реалізації патерна Factory Method було створено абстрактний клас MessageFactory.

У класі визначено фабричний метод createMessage(), який відповідає за створення об'єктів повідомлень.

Також реалізовано метод deliver(), який використовує створений об'єкт для відправлення повідомлення.

class MessageFactory {
public:
    virtual Message* createMessage() = 0;

    void deliver(const string& text) {
        Message* msg = createMessage();
        msg->send(text);
        delete msg;
    }
};
5. Створення конкретних фабрик

Для кожного типу повідомлення було створено окрему фабрику:

EmailFactory;
SmsFactory;
PushFactory.

Кожна фабрика перевизначає метод createMessage() та створює відповідний об'єкт повідомлення.

Наприклад:

class EmailFactory : public MessageFactory {
public:
    Message* createMessage() override {
        return new EmailMessage();
    }
};
6. Реалізація клієнтського коду

У головній функції main() створюються об'єкти фабрик та викликається метод deliver() для відправлення повідомлень.

Клієнтський код працює тільки з абстрактним класом MessageFactory і не створює об'єкти повідомлень безпосередньо.

MessageFactory* factory = new EmailFactory();
factory->deliver("Welcome to Factory Method");
7. Тестування програми

Після реалізації було виконано тестування для всіх типів повідомлень:

перевірено створення EmailMessage;
перевірено створення SmsMessage;
перевірено створення PushMessage;
перевірено правильність роботи методу send().

Усі тестові сценарії були виконані успішно.

8. Результат

У результаті створено програмне забезпечення для відправлення повідомлень із використанням патерна Factory Method. Реалізація дозволяє легко додавати нові типи повідомлень шляхом створення нового класу повідомлення та відповідної фабрики без зміни існуючого коду системи.
